Aprendendo o DNS Domain Name System.

O que é DNS

A tradução de DNS é Domain Name System, Sistemas de nomes de domínio. O DNS é como uma lista de telefones do google, para cada site na internet existe um registro, como nome é endereço. O serviço de DNS é bem simples quando navegamos na internet a procura de um site especifico como google.com, ele pega o nome que fornecemos e o converte para IP(Internet Protocol). O DNS foi feito para facilitar e automatizar a busca na internet. Nós seres humanos podemos memórizar muitas coisas, porem um endereço IP 187.217.5.110 é difícil, então o existe essa tradução de nome de domínios para IP. Sendo mais facil para nós humanos memórizar www.google.com Mas o www com o .com são o que?
A rede de alcance mundial World Wide Web(www) é um sistema interligado entre outros recursos, como exemplo documentos, imagens e videos. Que acessamos na internet. E o .com seria há extensão do site.


Servidor DNS Recursivo

Um servidor DNS recursivo é aquele que recebe a consulta de DNS do seu computador e resolve essa consulta, buscando a informação por meio de outros servidores DNS até encontrar a resposta correta.
como funcionas? Quando você digita no navegador, como "google.com", seu computador pergunta para o servidor DNS recursivo qual o endereço IP do site. Se o servidor recursivo não souber, ele vai:

  • Perguntar para o servidor DNS Raiz.

  • Depois perguntar para os servidores responsáveis pelos domínios de topo(".com", ".org", etc).

  • E, por fim, consultar o servidor DNS Autoritativo para aquele domínio especifico (como o servidor DNS de "google.com") para obter a responsta final (o IP).


Servidor DNS Autoritativo

O servidor DNS Autoritativo é o servidor que tem a informação final sobre um determinado nome de domínio. Ou seja, ele "sabe" a resposta correta para a consulta de um domínio específico. Existem dois tipos de servidores DNS autoritativo:

  • Servidor DNS autoritativo primário: É o servidor principal que contém as informações mais atualizadas e precisas sobre os domínios que ele é responsável.

  • Servidor DNS autoritativo secundário: Ele mantém uma copia das informações do servidor primário e serve como backup, garantindo que, se o servidor primário falhar, a consulta ainda será respondida.

Quando o seu computador ou servidor DNS faz uma consulta para encontrar o IP de um site, ele pode acabar sendo direcionado para DNS autoritativo. Esse servidor é o "dono" da informação sobre aquele domínio especifíco. Por exemplo, se você estiver procurando por "google.com", o servidor DNS autoritativo para "google.com" vai te dar a resposta exata, como o endereço IP do site.


Servidor DNS Raiz

O servidor DNS raiz é o primeiro servidor no processo de resolução de DNS. Ele não sabe o endereço IP exato de um site, mas sabe quem tem essa informação. Ele "guia" o processo de consulta para os servidores responsáveis pelos domínios de topo, que é chamado TDL - Top Level Domain verifica quem é responsável pelos domínios (".com", ".net", ".org", etc). E ele te redireciona para o servidor DNS autoritativo.


Servidor DNS de Cache

Um servidor DNS de cache armazena (ou "cacha") as resposta de consultas DNS anteriores por um determinado tempo. Isso ajuda a acelerar o processo de resolução de DNS. Quando você acessa um site pela primeira vez, o servidor DNS de cache armazena o IO do site po um tempo (tempo de expiração).


Servidor DNS Recursivo Inverso

O DNS recursivo inverso é o processo contrário do DNS normal. Ele tenta descobrir o nome de domínio a partir de um endereço IP. Ou seja, em vez de traduzir um nome (como "google.com") para um IP, ele faz o processo inverso.


Servidor DNS de Forwarding(ou servidor de Encaminhamento)

Um servidor DNS de forwarding é um tipo de servidor que encaminha as consultas para outros servidores DNS, em vez de resolver as consultas diretamente. Ele basicamente funciona como um intermediário que repassa as consultas para um servidor recursivo ou autoritativo.
