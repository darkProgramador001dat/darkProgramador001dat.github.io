Aprendendo o HTTP Hypertext Transfer Protocol

O que é HTTP

HTTP (Hypertext Transfer Protocol) é um protocolo de comunicação usado na web para tranferir e receber informações entre um cliente (geralmente um navegador, como Chrome ou Firefox) e um servidor web. Quando você acessa um site, o seu navegador faz uma solicitação HTTP ao servidor que hospeda o site, pedindo para carregar a página solicitada.

  • Requisição HTTP: quando você digita um endereço na barra de endereços do navegador (como "www.google.com"), o navegador envia uma solicitação HTTP para o servidor onde o site está hospedado. Essa solicitação inclui informações sobre o que o navegador deseja (por exemplo, uma página HTML, uma imagem, etc).

  • Resposta HTTP: o servidor responde a essa solicitação enviando os dados necessários de volta ao navegador. Isso pode ser uma página web, uma imagem ou outros arquivos. A resposta também inclui um código de status, que indica se a requisição foi bem-sucedida (como "200 OK") ou se houver algum erro (como "404 Not Found", ou "500 Internal Server Error").

Códigos de Status HTTP

os códigos de status indicam o resultado da requisição e são divididos em várias categorias:

  • 1xx (informativos): indicam que a solicitação foi recebida e o processo está em andamento.
    Exemplo: 100 Continue - O servidor recebeu a solicitação e o cliente pode continuar.

  • 2xx (Sucess): indica que a solicitação foi bem-sucedida. Exemplo: 200 OK - A solicitação foi bem-sucedida, e a resposta contém o recurso solicitado.
    Exemplo: 201 Created - A solicitação foi bem-sucedida, e um novo recurso foi criado.

  • 3xx (redirecionamento): indicam que o cliente precisa tomar uma ação adicional para completar a requisição (geralmente, um redirecionamento).
    Exemplo: 301 Moved Permanently - o recurso foi movido para outro lugar permanentemente.
    Exemplo: 302 Found - o recurso foi temporariamente movido para outra URL.

  • 4xx (Erros do Cliente): indica que houve um erro na solicitação feita pelo cliente.
    Exemplo: 400 Bad Request - a solicitação foi malformada ou contém erros.
    Exemplo: 403 Forbidden - o cliente não tem permissão para acessar o recurso.
    Exemplo: 404 Not Found - o recurso solicitado não foi encontrado no servidor.

  • 5xx (Erros do Servidor): indicam que o servidor falhou ao processar a solicitação.
    Exemplo: 500 (Internal Server Error) - o servidor encontrou um erro e não pode completar a requisição.
    Exemplo: 502 Bad Gateway - o servidor agiu como um gateway ou proxy e recebeu uma resposta inválida de um servidor upstream.


Métodos HTTP

Os métodos HTTP indicam o tipo de operação que o cliente deseja realizar no servidor. Os métodos mais comuns são:

  • GET: solicitação um recurso do servidor, como uma página HTML ou imagem. Este é o método mais usado.

  • POST: envia dados ao servidor (como quando você preenche um formulário de cadastro). É usado quando você deseja criar ou atualizar algo no servidor.

  • PUT: substitui um recurso existente no servidor. É semelhante ao POST, mas geralmente é usado para atualizar recursos.

  • DELETE: solicita que um recurso seja excluido do servidor.

  • HEAD: retorna apenas os cabeçalhos da resposta, sem o corpo. Pode ser útil para verificar se um recurso está disponivel sem precisar baixar todo o conteúdo.

  • OPTIONS: informa mais métodos HTTP o servidor suporta para um determinado recurso.

  • PATCH: modifica parcialmente um recurso no servidor, sem substituí-lo totalmente (diferente do PUT).



Cookie e Sessões

O HTTP é um protocolo sem estado, o que significa que cada requisição é independente. Para manter o contexto entre requisições (como quando você está logado em um site), usamos cookies e sessoẽs.

  • Cookie são pequenos arquivos armazenados no navegador do cliente que permitem que o servidor "lembre" informações entre requisições, como preferências de usuário ou estado de login.

  • Sessões funcionam de maneira similar, mas as informações são armazenadas no servidor e referenciadas por um identificador (ID) enviado para o cliente, geralmente através de um cookie.
